# iPad卡片显示问题修复

## 🐛 问题描述

在iPad上测试时发现卡片内容被截断：
- ❌ 顶部标题和图标没有显示
- ❌ 卡片内容不完整
- ❌ 底部按钮和文字被遮挡
- ❌ 整体布局显示不正常

## 🔍 问题分析

### 根本原因
1. **渐变遮罩位置不当**
   - ScrollView的mask从顶部就开始应用
   - 遮罩范围过大（0-0.88），导致内容被遮挡

2. **顶部间距不足**
   - iPad和iPhone使用相同的顶部padding（8px）
   - iPad需要更多空间来显示5列布局

3. **VStack间距固定**
   - 标题和ScrollView之间间距固定为20px
   - iPad需要更大的间距

## 🔧 修复方案

### 1. 调整渐变遮罩范围

**修改前：**
```swift
.mask(
    LinearGradient(
        gradient: Gradient(stops: [
            .init(color: .black, location: 0),
            .init(color: .black, location: 0.80),  // 80%处开始淡出
            .init(color: .clear, location: 0.88)   // 88%处完全透明
        ]),
        startPoint: .top,
        endPoint: .bottom
    )
)
```

**问题：**
- 遮罩在80%位置就开始淡出
- 导致底部20%的内容被遮挡
- 影响卡片的完整显示

**修改后：**
```swift
.mask(
    LinearGradient(
        gradient: Gradient(stops: [
            .init(color: .black, location: 0),
            .init(color: .black, location: 0.85),  // 85%处开始淡出
            .init(color: .clear, location: 0.95)   // 95%处完全透明
        ]),
        startPoint: .top,
        endPoint: .bottom
    )
)
```

**改进：**
- 遮罩在85%位置才开始淡出
- 只有底部10%的内容有淡出效果
- 更多内容可以完整显示

### 2. 增加顶部间距

**修改前：**
```swift
.padding(.top, 8) // 所有设备统一8px
```

**修改后：**
```swift
.padding(.top, horizontalSizeClass == .regular ? 20 : 8) // iPad 20px, iPhone 8px
```

**改进：**
- iPad顶部间距增加到20px
- 为5列布局提供更多呼吸空间
- iPhone保持原有8px不变

### 3. 调整VStack间距

**修改前：**
```swift
VStack(spacing: 20) {
```

**修改后：**
```swift
VStack(spacing: horizontalSizeClass == .regular ? 25 : 20) {
```

**改进：**
- iPad标题和内容区间距增加到25px
- 视觉层次更清晰
- iPhone保持20px不变
