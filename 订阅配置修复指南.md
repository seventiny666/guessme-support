# 订阅配置修复指南 - 元数据丢失问题

## 问题描述
在 App Store Connect 订阅界面显示"元数据丢失"

## 原因分析
"元数据丢失"通常是因为：
1. ❌ 订阅的本地化信息未填写
2. ❌ 订阅显示名称缺失
3. ❌ 订阅描述缺失
4. ❌ 未添加至少一种语言的本地化

## 修复步骤

### 第一步：访问订阅配置页面
1. 登录 App Store Connect
2. 访问：https://appstoreconnect.apple.com/apps/6759666661/distribution/subscription-groups/21954022
3. 或者：我的 App -> GuessMe -> 订阅 -> 订阅群组

## 常见错误和解决方案

### 错误1：元数据丢失
**原因**：未添加本地化或本地化信息不完整  
**解决**：添加至少一种语言的完整本地化信息

### 错误2：订阅无法保存
**原因**：必填字段未填写  
**解决**：检查所有必填字段（显示名称、描述）

### 错误3：价格未设置
**原因**：未选择价格等级或地区  
**解决**：设置价格并选择所有地区

### 错误4：审核被拒
**原因**：订阅信息不清晰或截图不符合要求  
**解决**：提供清晰的订阅说明和高质量截图

## 验证步骤

### 1. 在 App Store Connect 中验证
- 所有字段显示绿色对勾 ✓
- 没有"元数据丢失"警告
- 状态显示"准备提交"

### 2. 在 Xcode 中验证
检查 Products.storekit 文件：
```json
{
  "identifier": "com.seventinygame.app.yearly",
  "type": "auto-renewable",
  "displayName": "专业版年度订阅",
  "description": "解锁全部功能",
  "price": 1.99,
  "familyShareable": false,
  "subscriptionDuration": "P1Y",
  "introductoryOffer": {
    "type": "free-trial",
    "duration": "P3D"
  }
}
```

### 3. 在 App 中测试
1. 运行 App
2. 点击"升级到专业版"
3. 验证订阅信息显示正确
4. 使用沙盒账号测试购买流程

## 时间线

1. **配置订阅**：15-30 分钟
2. **上传截图**：5-10 分钟
3. **填写审核信息**：10-15 分钟
4. **验证配置**：5 分钟
5. **提交审核**：2 分钟

**总计：约 40-60 分钟**

## 注意事项

1. ⚠️ 订阅配置保存后，某些信息无法修改（如产品 ID）
2. ⚠️ 价格修改需要审核通过后才能生效
3. ⚠️ 免费试用只对新用户有效
4. ⚠️ 订阅必须在 App 提交审核前配置完成
5. ⚠️ 确保 App 代码中的产品 ID 与 App Store Connect 一致

4. 查看 App Store Connect 的错误提示
