## ✅ 最终解决方案

### 布局结构

```swift
NavigationView {
    GeometryReader { geometry in
        ZStack {
            // 1. 背景图片
            Image("background")
            
            // 2. 主内容VStack
            VStack(spacing: 0) {
                // 顶部标题栏
                HStack { 语言按钮 + 标题 + 规则按钮 }
                    .frame(height: iPad ? 160 : 100)
                
                // 中间卡片区域
                CardGridView(...)
                
                Spacer(minLength: 0)
                
                // 底部占位区域
                Color.clear.frame(height: iPad ? 160 : 140)
            }
            
            // 3. 底部按钮（绝对定位）
            VStack { 按钮内容 }
                .position(
                    x: geometry.size.width / 2,
                    y: geometry.size.height - (iPad ? 70 : 100)
                )
                .zIndex(9999)
            
            // 4. 隐藏的NavigationLink
            NavigationLink(...)
        }
    }
    .ignoresSafeArea(.all)
}
```

### 关键技术点

1. **GeometryReader** - 获取准确的屏幕尺寸
2. **VStack结构** - 保持顶部内容在正常流中
3. **底部占位区域** - 为绝对定位的按钮留出空间
4. **绝对定位底部按钮** - 确保始终显示在底部
5. **渐变背景** - 提供更好的视觉效果

## 💡 技术经验总结

### 为什么这个方案成功

1. **GeometryReader提供准确尺寸**
   - 不依赖UIScreen.main.bounds
   - 考虑了SafeArea和其他因素

2. **VStack保持正常流**
   - 顶部内容在正常布局流中
   - 不需要复杂的绝对定位计算

3. **底部占位区域**
   - 防止卡片区域被底部按钮遮挡
   - 确保滚动时有足够空间

4. **绝对定位底部按钮**
   - 始终固定在底部
   - 不受其他内容影响

### SwiftUI布局最佳实践

1. **优先使用正常布局流** - 只在必要时使用绝对定位
2. **GeometryReader获取尺寸** - 比UIScreen更准确
3. **分离iPad和iPhone逻辑** - 使用horizontalSizeClass判断
4. **添加占位区域** - 为绝对定位元素预留空间
