# 完整部署步骤 - 从零开始

## 当前情况
- ✅ 文件已准备好
- ✅ Git 本地仓库已配置
- ❌ GitHub 仓库不存在（需要创建）

## 方案1：在 GitHub 网页创建仓库（最简单）

### 第一步：创建 GitHub 仓库

1. 访问：https://github.com/new
2. 填写信息：
   - Repository name: `iosguessme`
   - Description: `你比划我猜 iOS App`
   - **选择 Public**（必须公开才能使用 GitHub Pages）
   - **不要**勾选 "Add a README file"
   - **不要**勾选 "Add .gitignore"
   - **不要**选择 License
3. 点击 "Create repository"

### 第二步：推送代码

创建仓库后，GitHub 会显示命令，在终端执行：

```bash
git push -u origin master
```

如果提示输入密码，使用 Personal Access Token（见下文）。

### 第三步：启用 GitHub Pages

1. 在仓库页面，点击 "Settings"
2. 左侧菜单找到 "Pages"
3. Source 选择：`Deploy from a branch`
4. Branch 选择：`master` 和 `/ (root)`
5. 点击 "Save"
6. 等待 1-5 分钟，页面会显示：
   ```
   Your site is live at https://seventiny666.github.io/iosguessme/
   ```

### 第四步：验证部署

访问：
- https://seventiny666.github.io/iosguessme/support-page/privacy.html
- https://seventiny666.github.io/iosguessme/support-page/

## 创建 Personal Access Token

如果推送时需要密码：

### 步骤：
1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" -> "Generate new token (classic)"
3. 填写：
   - Note: `iosguessme`
   - Expiration: `90 days`
   - 勾选：`repo` (所有子选项)
4. 点击 "Generate token"
5. **立即复制 token**（只显示一次）

### 使用 Token：
```bash
git push -u origin master
# Username: seventiny666
# Password: 粘贴你的 token
```

### 保存 Token（可选）：
```bash
# 保存凭据，下次不用再输入
git config --global credential.helper store
git push -u origin master
# 输入一次后会保存
```

## 常见问题

### Q: 推送时一直要求输入密码
A: 使用 Personal Access Token，并执行 `git config --global credential.helper store` 保存凭据

### Q: 提示 "Repository not found"
A: 先在 GitHub 网页创建仓库

### Q: GitHub Pages 显示 404
A: 等待 5-10 分钟，或检查 Settings -> Pages 是否正确配置

### Q: 页面显示但样式不对
A: 检查文件路径，确保 privacy.html 在 support-page 文件夹中
