  - iPhone: 60px → 100px (+40px)

- 在iPad大屏幕上布局更加舒展
- 操作体验更好

### 2. 退出弹窗按钮高度优化

#### 调整内容：
- **取消按钮高度+4px**: 
  - iPad: 54px → 58px (+4px)
  - iPhone: 48px → 52px (+4px)
- **退出按钮高度+4px**: 
  - iPad: 54px → 58px (+4px)
  - iPhone: 48px → 52px (+4px)

#### 代码变更：
```swift
// 取消按钮
.frame(height: horizontalSizeClass == .regular ? 58 : 52)

// 退出按钮
.frame(height: horizontalSizeClass == .regular ? 58 : 52)
```

#### 效果：
- 按钮更高，更容易点击
- 视觉上更加突出
- 与其他界面按钮高度更协调

### 3. 创建类型页面显示问题修复

#### 问题描述：
创建类型页面出现了和之前首页一样的问题：
- 内容显示不全
- 可能是SafeArea限制导致的布局问题

#### 修复方案：
应用与首页相同的修复策略：

1. **忽略SafeArea限制**：
```swift
.ignoresSafeArea(.all)
```

2. **优化ScrollView布局**：
```swift
// 减少底部padding，为底部内容留空间
.padding(.bottom, 30)  // 从100px减少到30px

// 添加Spacer确保布局正确
Spacer(minLength: 0)
```

3. **保持VStack结构**：
- 顶部导航栏（固定）
- 中间内容区（可滚动）
- 底部自动填充（Spacer）

#### 代码变更：
```swift
var body: some View {
    ZStack {
        // 背景
        Image("background")
            .ignoresSafeArea()
        
        VStack(spacing: 0) {
            // 顶部导航栏（固定）
            ZStack { ... }
            
            // 内容区域（可滚动，限制高度）
            ScrollView {
                VStack { ... }
                .padding(.bottom, 30)  // 减少padding
            }
            
            // 底部自动填充
            Spacer(minLength: 0)
        }
    }
    .ignoresSafeArea(.all)  // 关键修复
}
```
