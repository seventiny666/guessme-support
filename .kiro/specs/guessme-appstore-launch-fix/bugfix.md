# GuessMe App Store上线阻碍问题修复规范

## 介绍

GuessMe iOS应用在App Store上线前存在多个关键问题，这些问题将直接阻碍应用成功通过App Store审核和正常运行。主要问题包括：产品ID不匹配导致订阅功能无法工作、邮箱配置不一致、App Store Connect配置不完整等。这些问题如果不系统性解决，将导致应用无法成功上线或用户体验严重受损。

## Bug分析

### 当前行为（缺陷）

1.1 当订阅功能被触发时，系统使用错误的产品ID（com.guessme.app.*）与App Store Connect中配置的产品ID（com.guessme.app.*）不匹配，导致订阅功能完全无法工作

1.2 当用户查看不同文档和页面时，系统显示不一致的邮箱地址（784430005@qq.com 和 seventiny007@126.com），造成用户困惑和联系方式混乱

1.3 当App Store审核团队检查应用时，系统缺少必要的App Store Connect配置（订阅产品详细配置、应用截图、完整描述），导致无法提交审核

1.4 当用户进行功能测试时，系统存在未经验证的订阅功能和真机测试不完整，可能导致隐藏的功能性bug

### 期望行为（正确）

2.1 当订阅功能被触发时，系统应当使用统一的正确产品ID（com.guessme.app.*），确保与App Store Connect配置完全匹配，订阅功能正常工作

2.2 当用户查看任何文档和页面时，系统应当根据用途显示正确的邮箱地址：开发者相关使用784430005@qq.com，用户支持相关使用seventiny007@126.com

2.3 当App Store审核团队检查应用时，系统应当具备完整的App Store Connect配置，包括所有订阅产品详细配置、高质量应用截图、完整应用描述，满足审核要求

2.4 当用户进行功能测试时，系统应当通过完整的订阅功能测试和真机测试验证，确保所有功能正常运行无bug

### 不变行为（回归预防）

3.1 当用户使用核心游戏功能时，系统应当继续保持10个游戏分类、500+词汇、时间选择（30s/60s/90s/120s）、计分系统等核心功能正常运行

3.2 当用户在不同设备上使用应用时，系统应当继续保持iPad 4列布局、iPhone 2列布局、响应式设计、流畅动画效果等UI/UX优化

3.3 当用户关注隐私保护时，系统应当继续保持完全离线运行、不收集个人信息、完整隐私政策等隐私保护特性

3.4 当用户使用订阅相关功能时，系统应当继续保持StoreKit 2实现、3天免费试用、家庭共享支持、恢复购买功能等已实现的订阅特性
